Rollover

You can make an image change when you point to it. This is called a rollover. You need two images, preferably with the same dimensions. One is the original image shown on the page and the other is the image that appears when you point to the original image. When you move your pointer away the original image will appear again. Suppose we have an image called "boreas-t.jpg" and a second image called "boreas-r.jpg". If you click on the image you will link to "http://en.wikipedia.org/wiki/Boreas_(painting)". The name associated with the image and rollovers is "bore". Save the four lines below in the file as demoroll.html.

<a href="http://en.wikipedia.org/wiki/Boreas_(painting)" 
onmouseover="document.bore.src='boreas-r.jpg'" 
onmouseout= "document.bore.src='boreas-t.jpg'">
<img src="boreas-t.jpg" name="bore" alt="boreas" title="Boreas"> </a>

If you have another example on the same page, you cannot use the same name. See next example where the name is "circ". Put these 4 lines also in demoroll.html. Note: If you put this through Dreamweaver, it will not understand it and make it not work. 

<a href="http://en.wikipedia.org/wiki/Circe_Offering_the_Cup_to_Ulysses" 
onmouseover="document.circ.src='circe-r.jpg'" 
onmouseout= "document.circ.src='circe-t.jpg'">
<img src="circe-t.jpg" name="circ" alt="Circe" > </a>

Image Map

An image may be divided into areas called hotspots. Hotspots are links to another document that appears when you click on the hotspot. Hotspots may have the shape of a rectangle, polygon, or circle (as shown in the example below). The coordinates of a rectangle are specified by coords="x1, y1, x2, y2" where x1, y1 are the coordinates of the upper left corner of the rectangle, and x2, y2 and the coordinates of the lower right corner of the rectangle. The coordinates of a polygon are specified in a similar manner with each pair of coordinates being a vertex of the polygon. The coordinates of a circle are specified by coords="x, y, r" where x,y are the coordinates for the center of the circle and r is the radius. The map starts with the <map> tag, which has a name and id attribute, both of which specifies the name of the map. The <area> tag has a title attribute that displays when you move your pointer over the map, a shape attribute (rect for rectangle, poly for polygon, or circle for circle), and a href attribute that displays the specified document if you click on the hotspot. You can have as many hotspots in an image as you like. All area tags for an image are placed within the opening <map> and closing </map> tags. You can use Photoshop and click on Windows > Show Info or F8 to find the coordinates of the point you want to use. Just make sure you click on the arrow on the bottom right corner of the crosshair and set the units to pixels (not inches).

Copy the following lines of code and save it as demomap.html.

<!doctype html>
<html>
<head>
<meta charset="utf-8">
<title>Image Map</title>
<style type="text/css">
body,td,th {
font-family: Verdana, Geneva, sans-serif;
font-size: 14px;
}
body {
background-color: #FFC;
}
</style>
</head>
<body>
<h2>Image Map</h2>
<img src="bichon.jpg" usemap="#Map" title="Patty">
<map name="Map">
<area shape="circle" coords="83,57,12" href="circe.html" title="Left Eye">
<area shape="rect" coords="12,41,35,96" href="bark.wav" title="Right Ear">
<area shape="poly" coords="131,51,144,75,135,105,120,111,111,92,117,69,131,52" href="harp.wav" title="Left Ear">
</map>
<p>This is Patty.</p>
<ul>
<li>Click on his right ear to hear Marshmallow bark.</li>
<li>Click on his left eye to see Circe who was almost successful in keeping Odysseus from seeing his faithful dog again.</li>
<li>Click on his left ear to hear some harp music.</li>
</ul>
</body>
</html>

In the image tag <img> you must set the usemap attribute to the name specified by the name attribute of the <map> tag prefixed with the # sign. bichon.jpg is the image with the desired hotspots.

Forms

A form allows your users to send information to you (the programmer or script-writer who analyzes the information submitted). HTML merely creates the appearance of a form. In order for the form to work properly, you must create a script that runs on the server.

Create a folder called formsample inside the classwork/day7 folder. Save the following as demoform.html inside the formsample folder. 

<!doctype html>
<html>
<head>
<title>Tell Me Your Name</title>
</head>
<body>
<h3>Who are you?</h3>
<form method="post" action="http://users.fitnyc.edu/cgi-bin/post-query">
	<p>Enter your name:
		<input type="text" name="theName" size="35"></p>
	<p>Enter your zip code:
		<input type="text" name="theZip" size="10"></p>
	<p>Student:
		<input type="radio" name="theStudent" value="student_yes">Yes
		<input type="radio" name="theStudent" value="student_no">No</p>
	<p>Enter your email address:
		<input type="text" name="theEmail" size="27"></p>
	<p>Enter your comments or questions:<br>
		<textarea name="theComments" rows="3" cols="40" wrap="soft"></textarea></p>
	<p><input type="submit" value="Send Now"><input type="reset" value="Reset"></p>
</form>
</body>
</html>

The opening tag <form> and the closing tag </form> enclose the form. The method attribute indicates how the form input is given to the script that processes the form. Possible values are get and post. The action attribute indicates the CGI script used to process the form input. The prompt in "Enter your name" tells the user to enter information into the text box in the next line using the input tag. When the type attribute is "text", it can be omitted. The name attribute together with the value in the text box are sent to the CGI script as a pair. The size attribute is the width of the text box in number of characters. The default size is 20 characters. The next two lines show another example of a text box for input. The prompt "Student" shows an example where radio buttons are used. When the user chooses one radio button, the other(s) are automatically turned off. The next two lines show another example of using a text box for input. The next two lines show the use of a textarea tag. The textarea tag allows more than one line of text to be entered while the text box is only one line of input. The textarea is set with the number of rows and columns indicated by the rows and cols attributes respectively. wrap="soft" means that word wrap is turned on but is sent to the CGI script as one contiguous line of information. wrap="hard" also means that word wrap is turned on but information sent to the CGI script includes carriage returns and line feed characters where words were wrapped. The last line shows how a submit and a reset button are created.

Here is a coding example for special characters:

<!doctype html>
<html>
<head>
<title>Special Characters</title>
</head>
<body>
<h3>Special Characters</h3>
<p>copyright symbol: &copy;</p>
<p>code: </p>
<pre>
&lt;p&gt;copyright symbol: &amp;copy;&lt;/p&gt;
</pre>
<hr>
<p>a red heart: <span style="color:red;">&hearts;</span></p>
<p>code:</p>
<pre>&lt;p&gt;a red heart: &lt;span style=&quot;color:red;&quot;&gt;&amp;hearts;&lt;/span&gt;&lt;/p&gt;</pre>
<p>You can also write the code like this:</p>
<pre>&lt;p&gt;a heart: ♥&lt;/p&gt;</pre>
<p>but it won't be red.</p>
<hr>
</body>
</html>

Meta Tag

Meta tags are placed inside the <head> tag.

  • Keywords
<meta name="keywords" content="Web construction, Web, HTML, sounds,
 images, frameset, frames, tables" /> 

  • Description
<meta name="description" content="An example website for first level
 students taking a course in HTML and an editor such as Dreamweaver." />  

  • Author
<meta name="author" content="Gene Chin e-mail:" />  

  • Expiration Date
<meta name="expires" content="Mon, 31 December 2007 05:00:00 GMT" />  

  • Robots: Allow search engines to read page and follow all links
<meta name="robots" content="all, follow" />  

  • Robots: Allow search engines to read page but don't follow links
<meta name="robots" content="all, nofollow" />  

  • Robots: Prevent search engines from reading your personal pages
<meta name="robots" content="noindex, nofollow" /> 

  • Robots: Keep search engines from archiving your pages
<meta name="robots" content="noarchive" />  

  • Refreshing Page Content every 7 seconds
<meta http-equiv="refresh" content="7" />  

  • Refreshing Page Content and load another document after 7 seconds
<meta http-equiv="refresh" content="7;url=http://www.w3c.org" />  

  • Character encoding  
pre-HTML5: <meta http-equiv="content-type" content="text/html; charset=UTF-8" />
HTML5:     <meta charset="utf-8">
