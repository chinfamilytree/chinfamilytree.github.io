Fonts

The font and basefont tags are deprecated in HTML 4.01 and XHTML. The World Wide Web Consortium has removed both tags from its list of recommendations. Some people are still using them but you should avoid using them and use styles instead. Using styles you can set the font-family (was face), font-size (was size), color and other properties. Here is an inline example of styles for fonts:

<!doctype html>
<html>
<head>
<title>Sample Inline Style for Fonts</title>
<meta charset="utf-8">
</head>
<body>
<h1 style="font-family:verdana, arial; font-size:500%; color:blue;">Styles define</h1>
<p style="font-family:georgia; font-size:36pt; color:red;">how to display (36pt) MMMM</p>
<p style="font-family:georgia; font-size:36px; color:navy;">how to display (36px) MMMM</p>
<p style="font-family:impact; font-size:2em; color:green;">HTML elements. (2em) MMMM</p>
<p style="font-family:times new roman; font-size:1em; color:green;">Times New Roman (1em) MMMM</p>
<p style="font-family:times new roman; font-size:16px; color:green;">Times New Roman (16px) MMMM</p>
<p style="font-family:arial; font-size:1em; color:green;">Arial (1em) MMMM</p>
<p style="font-family: segoe ui; font-size:1em; color:green;">Segoe UI (1em) MMMM</p>
<p style="font-family: segoe ui; font-size:16px; color:green;">Segoe UI (16px) MMMM</p>
<p style="font-family:comic sans ms; font-size:36px; color:orange;">Isn't this great? MMMM</p>
<p style="font-family:impact; font-size:36px; color:#e3001b;">Avoid using inline styles unless there is a special need!</p>
</body>
</html>

If you have more than one value for the font-family property use commas to separate them. The font-size property may be expressed in percents (%), points (pt), pixels (px), medium, large, small, and more. Using pixels is better across platforms for the screen. Use points for printing a document. The default size for a paragraph, <p>, is 16px, which is equal to 1em. The W3C recommends using em for font size. The color property may be a color name (e.g. red) or hex number (e.g. #ff0000).

Common fonts between Windows OS and Mac OS are: Arial, Arial Black, Comic Sans MS, Courier New, Georgia, Impact, Symbol ( abg) , Times New Roman, Trebuchet MS, Verdana, and Webdings ( 3456 ). See an example of these fonts.

Tables

Tables allow you to present information in rows and columns (tabular data). Tables should no longer be used to divide a page into sections.

  1. Opening and closing tables tags are <table> and </table> respectively. Practically none of the former particular attributes for <table> are supported in HTML5. Use CSS instead.
  2. A table row, whose opening and closing tags are <tr> and </tr>, is enclosed inside the table tags. All former attributes particular for <tr> are not supported in HTML5. Use CSS instead.
  3. Inside the table row, you can have a table header cell: <th> or a regular table data cell <td>. Their respective closing tags are </th> and </td>. No attributes that are particular to <th> or <td> are supported in HTML5. Use CSS instead.
  4. You can merge cells across columns by using the colspan attribute inside the opening <td> or <th> tags. For example: <th colspan="2"> would merge two columns. Use rowspan to merge rows in a similar manner, <td rowspan="2">. Note that the colspan and rowspan attributes are not available in CSS per se and are not stated as being supported by HTML5. Very convoluted solutions with nested <div> have been offered but these solutions are too complicated for this beginner course. I've tested rowspan and colspan in the major browsers (Chrome, Firefox, Internet Explorer, Opera, Safari) and W3C Markup Validator for HTML5. In all cases the 2 attributes worked fine.
  5. Use the <caption> tag to create a title or caption such as <caption>About My Friends and Me</caption>. The <caption> tag must immediately follow the <table> tag. Captions default to the top of the table and is centered across the width of the table. You can use the internal style sheet such as:
      <style type="text/css">
  caption {caption-side:bottom;}
  </style>
    to center the caption on the bottom of the table.
  6. A border can be created around a table and then you set its thickness (border-width:n), style (border-style:type, and color (border-color). The border-width may be in abbreviated units such as 2px. Notice that the border looks like a double border, one border for the table and another for the cells in the table. The type in the border-style:type property value pair is none, dotted, dashed, solid, double, groove, ridge, inset, or outset. The border-color may be one of 140 color names or a hexadecimal triplet. You can also specify the border properties all at once like this: border (-top, -right, -bottom, or -left) to make all 4 sides the same. "-property" is optional, where property is one of width, style, or color. Now type the ":" separator. Then type the appropriate values for each type. For example, a 1 pixel, solid, teal border would be specified as:
    table {border:1px solid teal;}
  7. To change the size of individual cells use the width and height properties for the th and td selectors. For example: 
    th {height:30px;}
  8. Background image image may be specified as follows:
    table {background-image: url("bg2.jpg");}

Sample Table

Use the following code to produce the table below and save as table1.html in your classwork/lesson5 folder.

<!doctype html>
<html>
<head>
<title>Tables</title>
<meta charset="utf-8">
<style type="text/css">
  body,td,th {
    font-family: Segoe ui;
    font-size: 1em; color:navy;}
  body {	background-color: #FFF;}
  table {border-collapse:collapse;
    border:solid teal 1px;
    width:500px;
    background-image: url("bg2.gif");}
  td,th {border:1px solid teal;}
  th {height:30px; vertical-align:middle;}
  td {padding:8px;}
</style>
</head>
<body>
  <table>
    <tr>
      <th rowspan="2"><img src="bg1.gif" alt="bg2"></th>
