HTML

Hypertext Markup Language (HTML) is the standard medium for publishing hypertext on the World Wide Web. It can be created and processed by a wide range of tools, from simple text editors such as Microsoft Notepad, Microsoft WordPad, TextEdit (on the Mac) to sophisticated editors such as Dreamweaver. HTML uses tags to provide content for headings, paragraphs, lists, links, etc.

↑Top

Tags

A tag tells a browser how to display content. A tag usually comes in pairs: an opening tag and a closing tag. An opening tag is always enclosed by angle brackets, such as <title>, which is the title tag. A closing tag is always enclosed by angle brackets with a slash immediately after the left angle bracket, such as </title>. Between the opening tag and the closing tag is text, called content. There are some tags, such as line break, <br />, that do not have content. These tags do not have a closing tag. We can also write this tag as <br>, <br/>, <BR>, <BR/> or <BR /> because HTML5 is backwards compatible with HTML 4.01 and XHTML 1.0. I will use the form which has the least amount of typing, <br>.

↑Top

White Space

White space (spaces, tabs, and hard returns) in an HTML document will be removed except for one space by a browser before it is displayed. Use white space to make your document more readable.

↑Top

Indentation of Code

Indentation of code in your document is not required by HTML. However, it makes your document much easier to read and debug.

↑Top

First Web Page

Every document starts with the document type tag. <!doctype html> is for HTML5. The document type specifies which standard is being followed by the html document and is used by validators, such as the W3C Markup Validator to check the markup validity of web documents. This tag does not have a closing tag. This line may be written in capital letters. Even though HTML5 is backwards compatible with HTML 4.01 (which uses capital letters) and XHTML 1.0 (which uses lowercase letters), I recommend that you use lower case letters in all your documents. Also use lowercase letters for your file names and extensions, e.g., contact.html or wpc.css.

↑Top

<html>
The second line of every document in HTML5 is the <html> tag and ends with the </html> tag. 

	<!doctype html>
	<html>
       :
       :
    </html>

<head>
The <head> tag is the document header and can provide information for users and searchers. The <head> tag must follow the opening <html> tag and precede the opening <body> tag.

	<!doctype html>
	<html>
       <head>
          :
       </head>
          :
          :
    </html>

<title>
The <title> tag is the name of your Web page and appears in the title bar when your HTML document is displayed by a browser. It goes within the <head>...</head> tags like this:

    <!doctype html>
    <html>
       <head>
          <title>Simple Web Page</title>
       </head>
          :
          :
       </html>

<body>
The <body> tag contains the bulk of the HTML document. It must follow the <head> tag and there can be only one <body> just as there can only be one <head>.

    <!doctype html>
    <html>
       <head>
          <title>Simple Web Page</title>
       </head>
       <body>
          :
          :
       </body>
    </html>

<h1> <h2> <h3> <h4> <h5> <h6>
The <h1> tag creates the largest heading and is of the greatest importance. The <h2> tag is a little less important than <h1> and <h6> is the least important and creates the smallest heading. All heading tags will produce bold text. Heading tags are block-level elements, that is, there is a line space before and after the heading when displayed by a browser.

<p>
The <p> tag creates a new paragraph and the </p> end the paragraph. The paragraph tag is also a block-level element.

Some Other Text Tags

<strong>...</strong>, <em>...</em>
Text within the <strong>...</strong> tags are bold, and within the <em>...</em> are italic. You can also use <b>...</b> for bold and <i>...</i> for italic. These tags are inline elements, that is, in the same line, as opposed to block-level elements that produce line spaces before and after. The heading tags, h1, h2, h3, h4, h5, and h6 are block-level elements. <p> is also a block-level element.

Preformatted text uses the <pre> and </pre> tags. Everything between these two tags will be displayed exactly as you have typed them on the page, including all white spaces. By default, preformatted text uses the "Courier New" font family. Changing the font face to a proportionally spaced font (not monospaced font) would destroy the look of a monospaced font.

Internal CSS

Use an internal (embedded) style sheet inside the head section to align a heading (default is left), change the background color (default is white), and change the default text color (default is black). A code sample follows:

<style type="text/css">
body {background-color:aqua; color:navy;}
h1 {text-align:center;}
p {font-family:verdana,arial,helvetica; 
  font-size:11px;}
</style>

The portion in front of the braces {} is called a selector. We will use only tags, for now, as our selectors. Enclosed in braces are property-value pairs with a colon (:) in between and separated from other property-value pair by a semicolon (;).

To align all headings substitute this line

h1,h2,h3,h4,h5,h6 {text-align:center;}

for the h1 line above.

Colors

Background color and text color may be any one of the 16 predefined colors (red #ff0000, lime #00ff00, blue #0000ff, aqua #00ffff, fuchsia #ff00ff, yellow #ffff00, black #000000, gray #808080, silver #c0c0c0, white #ffffff, navy #000080, green #008000, teal #008080, maroon #800000, purple #800080, olive #808000) or any of 16 million colors expressed in hexadecimal triplets. The first of the triplets represents 256 shades of red; the second triplet represents 256 shade of green; and the third triplets represents 256 shades of blue. The hexadecimal numbers vary from 00 (decimal 0) to FF (decimal 255). For example: red is "#ff0000", that is, all red, no green, and no blue. Hexadecimal triplets are case insensitive.

A color value may be one of 16 predefined color names or 145 color names in X11 color names (Some of the names in X11 color names may refer to the same color: e.g., fuchsia and magenta.) or a hex triplet, prefixed by a hash mark to represent any one of 16, 777, 216 colors.

A hex triplet uses the format: #rrggbb, where rr is the hexadecimal value for Red, gg is the hexadecimal value for Green and bb is the hexadecimal value for Blue. Note that Lime is the W3C designation for the RGB Green. which is different from the W3C green.

For example, the hex triplet "#800080" and "purple" both refer to the same color. An alternate name for aqua is cyan and magenta may be substituted for fuchsia. You could also use the names or hex triplets in X11 color names or the hex triplets in List of Colors. Note that some of the names in X11 color names and the List of Colors do not work. To be safe use the hex triplets in either X11 color names or List of Colors. When you use color names make sure you remove all spaces in the color names. This has been done in w3schools.com's color names, where you can view any of 141 colors as the background color along with different text colors by clicking on any of 141 color names (or a hex value). An HTML color picker is also available in w3schools.com where you can get lighter or darker shades of any color.

You can download a copy of RGB Color Model. This is an .exe file written by Gene M. Chin in 1999 and works on PCs only. The display is in hexadecimal and decimal. There are three sliders, each corresponding to the three primary additive colors: R, G, and B. Clicking on the left arrow of a slider decrements the value by 1. Clicking on the right arrow of a slider increments the value by 1. Clicking on a slider increments or decrements by 51 decimal (33 hexadecimal). You can drag a slider to any value you want to display all 16,777,216 colors, one color a time.

Encoding

A meta tag to show character encoding must be included in the head section such as:
<meta charset="utf-8">

UTF-8 is Unicode 8 bits. This encoding is for all languages supported by the W3C Consortium. This includes all European and Asian languages plus some other less known languages.

A Model Web Page

<!doctype html>
<html>
<!-- The following section is the <head> section, containing 
     a title, a meta tag for the character set utf-8, 
     and an internal (embedded) style sheet.
-->
<head>
<title>Model Web Page</title>
<meta charset="utf-8">
<style type="text/css">
body {
   background-color:bisque;
   color:firebrick;
}
h1 {
   text-align:center;
}
p {
   font-family:verdana; 
   font-size:14px;
}
</style>
</head>
<!-- The following section is the <body> section.
-->

   <h1>Web Page Construction Class</h1>

</html>
