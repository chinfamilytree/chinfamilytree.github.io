HTML/CSS Lesson 8 - Summary of CSS

External Style Sheets | Internal (Embedded) Style Sheets | Inline Styles | Hierarchy of Styles |

There are three types of cascading style sheets (CSS): external style sheet, internal style sheet (also called embedded style sheet), and inline styles. Style sheets are constructed using a simple text processor such as Notepad. Style sheets use braces as delimiters instead of angle brackets, which are used in HTML.

External Style Sheets

HTML was designed to show the content of a web page whereas a style sheet is used to show the presentation or format of a web page. An external style sheet is a separate plain text document with a name, such as css1.css, where css must be the file extension.

Here is an example of an external style sheet which must be linked to a page to effect the presentation of that page. The syntax is:

selector {property: value}      e.g. h1 {font-family: verdana;}
If you wish to specify more than one property:value pair, separate them with a semicolon (;). 
h1 {font-family: verdana; color: teal}
You can also group selectors so that the entire group will have the same properties. 
h1, h2, h3 {font-family: verdana; color: teal}
You must insert an external style sheet by using the <link> tag in the <head> section of the page. This has the same effect as if the external style sheet is in the <head> section and thus becomes an internal style sheet, explained below. The advantage comes in when you have two or more pages that use the same style sheet. Any change in the style sheet will effectively change every page that is linked to it. 
<head>
<link rel="stylesheet" type="text/css" href="css1.css" />
</head>

See an example of a link to a style sheet in line 6 of the code of ophelia.html. Note that the external style sheet has:
background-color: #FFC;
background-image: url(bg2.gif);

Internal (Embedded) Style Sheets

An internal style sheet (or embedded style sheet) is used when a single page has a unique style. The internal style sheet is defined in the <head> section of a document using the <style> tag. See an example of an internal style sheet in line 7 through line 27 in the code of ophelia.html

The external style sheet set the background-color and background-image to #FFC and url(bg2.gif) respectively. But both the background color and image are superceded by the internal style sheet (which physically comes after the link):
background-color: #C3FFE7;
background-image: url(bg1.gif);

If the internal style sheet physically came before the link, the background-color and background-image would not have changed. Note that three link colors have changed to a dark green (#060) and the hover color changed to "white" with "black" as the background color..

<head>
<meta charset="utf-8">
<title>John William Waterhouse</title>
<link rel="stylesheet" type="text/css" href="css1.css" />
<style type="text/css">
body {
background-color: #C3FFE7;
background-image: url(bg1.gif);
background-repeat: repeat-x;
}
a:link {
color: #060;
}
a:visited {
color: #060;
}
a:hover {
color: white;
background-color:black;
font-size:14px;
}
a:active {
color: #060;
}
</style>
</head>

Inline Styles

The following is an example of inline styles. The style attribute is used to change the default properties of a tag.

<html>
<body>
<h1 style="font-family:verdana; font-size:500%; color:blue;">Styles define</h1>
<p style="font-family:book antiqua; font-size:36pt; color:red;">how to display</p>
<p style="font-family:comic sans ms; font-size:36px; color:green;">HTML elements.</p>
</body> 
</html>

Hierarchy of Styles

The hierarchical order of precedence is:
  1. browser default - lowest priority
  2. external style sheet
  3. internal style sheet
  4. inline styles - highest priority

An exception to this rule is the background image which is tiled. The image always takes precedence over a background color.
